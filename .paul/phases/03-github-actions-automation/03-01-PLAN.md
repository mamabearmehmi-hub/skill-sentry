<objective>
## Goal
Create the reusable auditor workflow (the core muscle that all other workflows call) and the on-demand scan workflow (triggered by the Submit button via `repository_dispatch`). Also create a registry update script that merges audit results into `registry.json` and commits them back via `git-auto-commit-action`.

## Purpose
This is the pipeline that connects the auditor brain (Phase 2) to the Git-as-a-DB storage. Without this, audits only exist in memory. After this plan, a `repository_dispatch` event can trigger a full scan and auto-commit the results — the exact flow the Submit button (Phase 5) will trigger.

## Output
- `.github/workflows/reusable-auditor.yml` — reusable workflow: accepts repo URL, runs auditor, outputs JSON
- `.github/workflows/on-demand-scan.yml` — `repository_dispatch` listener, calls reusable auditor, commits results
- `scripts/update-registry.ts` — reads registry.json, upserts audit result, writes back
</objective>

    Create scripts/update-registry.ts that:
    
    1. Reads a RegistryEntry JSON from stdin (piped from cli.ts output)
       OR accepts a file path as argv[2] containing the JSON
    2. Reads public/data/registry.json
    3. Upserts the entry:
       - Find existing entry by matching owner + name
       - If found: replace it with the new entry

       - lastUpdated: new Date().toISOString()

    6. Logs: "Updated registry: {owner}/{name} — risk score {score}"

      npx tsx scripts/cli.ts $URL > /tmp/result.json
      npx tsx scripts/update-registry.ts /tmp/result.json

<task type="auto">
  <name>Task 2: Create reusable auditor workflow</name>
  <files>.github/workflows/reusable-auditor.yml</files>
  <action>
    Create .github/workflows/reusable-auditor.yml:
    
    ```yaml
    name: Reusable Auditor
    on:
      workflow_call:
        inputs:
          repo_url:
            description: 'GitHub repo URL to audit'
            required: true
            type: string
        outputs:
          audit_result:
            description: 'JSON audit result'
            value: ${{ jobs.audit.outputs.result }}
    
    jobs:
      audit:
        runs-on: ubuntu-latest
        outputs:
          result: ${{ steps.scan.outputs.result }}
        steps:
          - uses: actions/checkout@v4
          
          - uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'
          
          - run: npm ci
          
          - name: Run security audit
            id: scan
            run: |
              RESULT=$(npx tsx scripts/cli.ts "${{ inputs.repo_url }}")
              echo "result<<EOF" >> $GITHUB_OUTPUT
              echo "$RESULT" >> $GITHUB_OUTPUT
              echo "EOF" >> $GITHUB_OUTPUT
    ```
    
    Key points:
    - Uses workflow_call (reusable workflow pattern)
    - Pinned action versions (@v4 not @main)
    - npm ci for deterministic installs
    - Captures JSON output via GITHUB_OUTPUT multiline syntax
    - No secrets needed — auditor only uses git clone (public repos)
    
    Avoid:
    - Do NOT add deployment steps — this workflow only scans
    - Do NOT commit results — the calling workflow handles that
  </action>
  <verify>python3 -c "import yaml; yaml.safe_load(open('.github/workflows/reusable-auditor.yml'))" 2>&1 || echo "YAML validation requires PyYAML"</verify>
  <done>AC-1 satisfied: Reusable workflow accepts URL, runs CLI, outputs JSON. AC-4 partial: Valid YAML with pinned versions.</done>
</task>

<task type="auto">
  <name>Task 3: Create on-demand scan workflow</name>
  <files>.github/workflows/on-demand-scan.yml</files>
  <action>
    Create .github/workflows/on-demand-scan.yml:
    
    ```yaml
    name: On-Demand Scan
    on:
      repository_dispatch:
        types: [scan-repo]
    
    jobs:
      scan:
        uses: ./.github/workflows/reusable-auditor.yml
        with:
          repo_url: ${{ github.event.client_payload.repo_url }}
      
      update:
        needs: scan
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          
          - uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'
          
          - run: npm ci
          
          - name: Update registry
            run: |
              echo '${{ needs.scan.outputs.audit_result }}' > /tmp/result.json
              npx tsx scripts/update-registry.ts /tmp/result.json
          
          - name: Commit updated registry
            uses: stefanzweifel/git-auto-commit-action@v5
            with:
              commit_message: "audit: ${{ github.event.client_payload.repo_url }}"
              file_pattern: public/data/registry.json
    ```
    
    Key points:
    - Triggered by repository_dispatch type "scan-repo"
    - Extracts repo_url from client_payload (set by Submit button API route)
    - Calls reusable-auditor.yml to run the scan
    - Passes audit result to update-registry.ts
    - Uses git-auto-commit-action@v5 (pinned) to commit only registry.json
    - Commit message includes the scanned URL for traceability
    
    The Submit button (Phase 5) will trigger this via:
    octokit.rest.repos.createDispatchEvent({
      event_type: "scan-repo",
      client_payload: { repo_url: "https://github.com/..." }
    })
    
    Avoid:
    - Do NOT add Vercel redeploy trigger yet — Phase 5 handles that
    - Do NOT add error handling for invalid URLs — the auditor already throws
  </action>
  <verify>python3 -c "import yaml; yaml.safe_load(open('.github/workflows/on-demand-scan.yml'))" 2>&1 || echo "YAML validation requires PyYAML"</verify>
  <done>AC-2 satisfied: repository_dispatch listener, calls reusable auditor, updates registry, auto-commits. AC-4 satisfied: Valid YAML, pinned versions, secrets placeholder.</done>
