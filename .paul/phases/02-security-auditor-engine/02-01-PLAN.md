<objective>
## Goal
Build the core auditor engine — a stateless TypeScript module that accepts a GitHub repo URL, shallow-clones it to a temp directory, applies all 11 security rules from `lib/constants.ts` against matching files, computes a risk score, and returns a fully-typed `RegistryEntry` JSON report. This is the brain of Skill Sentry.

## Purpose
Every other feature (dashboard, submit button, daily scraper, report pages) depends on the auditor producing reliable, typed scan results. Getting this right is the single most important phase of the project. The auditor must be stateless (URL in → JSON out) so both the GitHub Action and the submit button can call the same function.

## Output
- `scripts/auditor.ts` — main entry: `auditRepo(url: string): Promise<RegistryEntry>`
- `scripts/clone-repo.ts` — shallow git clone to temp dir, cleanup after
- `scripts/scan-files.ts` — file walker + regex matcher against security rules
</objective>

  <action>
    Create scripts/clone-repo.ts that exports:
    
    `cloneRepo(url: string): Promise<{ repoDir: string; cleanup: () => Promise<void> }>`
    
    Implementation:
    - Validate URL is a GitHub repo (regex: github.com/{owner}/{repo})
    - Extract owner and name from URL
    - Create temp dir using Node.js `fs.mkdtemp` (in os.tmpdir())
    - Run `git clone --depth 1 {url} {tempDir}` using child_process.execSync
      - Set timeout of 30 seconds (abort hanging clones)
      - Capture stderr for error reporting
    - Return the temp dir path and a cleanup function that `rm -rf`s the temp dir
    - The cleanup function must work even if scanning throws
    - Export `parseGitHubUrl(url: string): { owner: string; name: string }` as a separate utility
    
    Avoid: 
    - Do NOT use any npm package for git operations (no simple-git, no isomorphic-git)
    - Use Node.js built-ins only (child_process, fs, path, os)
    - Do NOT catch errors silently — let them propagate with context
  </action>
  <verify>npx tsc --noEmit scripts/clone-repo.ts</verify>
  <done>AC-2 satisfied: Shallow clone with temp dir, cleanup guaranteed on error</done>

<task type="auto">
  <name>Task 2: Build scan-files.ts — file walker + regex matcher</name>
  <files>scripts/scan-files.ts</files>
  <action>
    Create scripts/scan-files.ts that exports:
    
    `scanRepository(repoDir: string): AuditFinding[]`
    
    Implementation:
    1. Walk the repo directory recursively using fs.readdirSync (recursive option)
       - Skip: node_modules/, .git/, dist/, build/, coverage/, .next/
       - Collect all file paths relative to repoDir
    
    2. For each SecurityRule in SECURITY_RULES:
       - Filter files matching the rule's fileTargets globs
         - "package.json" → exact match at root
         - "**/*.ts" → any .ts file in any directory
         - Use simple glob matching (no minimatch needed — just check extension + path)
       - Read each matching file's content
       - Apply the rule's regex pattern against the content
       - For each match, determine the line number:
         - Split content by newlines, find which line contains the match
       - Create an AuditFinding:
         - ruleId: rule.id
         - severity: rule.severity  
         - file: relative path from repo root
         - line: line number (1-indexed)
         - match: matched text, truncated to 200 chars
         - message: rule.description
         - score: rule.score
    
    3. Return all findings sorted by severity (CRITICAL first, then HIGH, then MEDIUM)
    
    Also export:
    `countFindingsBySeverity(findings: AuditFinding[]): { critical: number; high: number; medium: number; low: number; info: number }`
    
    Avoid:
    - Do NOT use glob/minimatch packages — implement simple matching inline
    - Do NOT read binary files (skip files > 1MB)
    - Do NOT follow symlinks
  </action>
  <verify>npx tsc --noEmit scripts/scan-files.ts</verify>
  <done>AC-3 satisfied: Rules applied to matching file targets, line numbers recorded. AC-5 partial: Clean scan logic exists.</done>
</task>

<task type="auto">
  <name>Task 3: Build auditor.ts — stateless orchestrator assembling the full pipeline</name>
  <files>scripts/auditor.ts</files>
  <action>
    Create scripts/auditor.ts that exports:
    
    `auditRepo(url: string): Promise<RegistryEntry>`
    
    Implementation:
    1. Parse URL with parseGitHubUrl(url) — get owner, name
    2. Clone repo with cloneRepo(url) — get repoDir, cleanup
    3. Wrap scanning in try/finally to guarantee cleanup:
       try {
         - Read package.json from repoDir (if exists) to extract:
           - description (from package.json description field)
           - packageName (from package.json name field)
         - Count scannable files (total files minus excluded dirs)
         - Run scanRepository(repoDir) to get findings
         - Compute riskScore: Math.min(MAX_RISK_SCORE, sum of finding scores)
         - Compute totalFindings using countFindingsBySeverity
         - Assemble RegistryEntry:
           - owner, name from parsed URL
           - url: the input URL
           - description: from package.json or null
           - stars: 0 (GitHub API metadata is NOT the auditor's job — Phase 3 enriches this)
           - topics: [] (same — enriched later)
           - lastAuditDate: new Date().toISOString()
           - riskScore, findings, totalFindings
           - verifiedSafe: riskScore === 0
           - scannedFiles: count
           - packageName: from package.json or null
       } finally {
         - await cleanup() — always runs
       }
    4. Return the RegistryEntry
    
    The function must be completely stateless:
    - No globals
    - No database reads/writes
    - No environment variable requirements (git must be in PATH, that's it)
    - Caller decides what to do with the result (write to registry, display, etc.)
    
    Avoid:
    - Do NOT write to registry.json — the auditor only produces data
    - Do NOT fetch GitHub API data (stars, topics) — that's Phase 3's scraper job
    - Do NOT import Octokit or any HTTP library
  </action>
  <verify>npx tsc --noEmit scripts/auditor.ts</verify>
