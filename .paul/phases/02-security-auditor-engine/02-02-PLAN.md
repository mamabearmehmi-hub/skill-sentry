<objective>
## Goal
Create a CLI runner for the auditor (so GitHub Actions can invoke it) and comprehensive integration tests with mock repo fixtures that prove the scanner actually catches malicious patterns and clears safe ones. This closes AC-5 and AC-6 from Plan 02-01.

## Purpose
The auditor engine exists but is unproven. Before any other phase builds on it, we need hard evidence it works: a CLI that GitHub Actions can call (`npx tsx scripts/cli.ts <url>`), and tests using local fixture directories that validate every severity level. The CLI is also how users can run audits locally.

## Output
- `scripts/cli.ts` — CLI entry point: accepts a GitHub URL, runs auditRepo, outputs JSON
- `scripts/__tests__/auditor.test.ts` — integration tests using local fixtures
- `scripts/__tests__/fixtures/` — safe and malicious mock repos
- Updated `package.json` with test script
</objective>
