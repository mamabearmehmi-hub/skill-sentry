<objective>
## Goal
Initialize the Next.js 15 project with Tailwind CSS and Shadcn UI, define the complete data contract (TypeScript interfaces), establish security heuristic constants with regex patterns, and seed the registry with mock entries — giving Phase 2+ a rock-solid foundation to build on.

## Purpose
Everything in Skill Sentry depends on the data contract. The auditor needs to know what findings look like. The UI needs to know what a registry entry contains. The GitHub Actions need to know what to commit. By defining types and constants first, we prevent integration mismatches across all 6 phases.

## Output
- Working Next.js 15 app (boots with `npm run dev`)
- `lib/types.ts` — all TypeScript interfaces
- `lib/constants.ts` — security heuristic patterns and scoring
- `public/data/registry.json` — seeded with 1 safe + 1 malicious mock entry
- Shadcn UI installed with base components
</objective>

<task type="auto">
  <name>Task 2: Define data contract (types) and security constants</name>
  <files>lib/types.ts, lib/constants.ts</files>
  <action>
    1. Create lib/types.ts with these interfaces:

       SeverityLevel: "CRITICAL" | "HIGH" | "MEDIUM" | "LOW" | "INFO"
       
       SecurityRule: {
         id: string (e.g., "CRIT-001")
         name: string (e.g., "Malicious Lifecycle Script")
         severity: SeverityLevel
         pattern: RegExp
         score: number (CRITICAL=100, HIGH=50, MEDIUM=20, LOW=10, INFO=0)
         description: string
         fileTargets: string[] (e.g., ["package.json"] or ["**/*.ts"])
       }
       
       AuditFinding: {
         ruleId: string (references SecurityRule.id)
         severity: SeverityLevel
         file: string (relative path where found)
         line: number | null (line number if available)
         match: string (the matched text, truncated to 200 chars)
         message: string (human-readable explanation)
         score: number
       }
       
       RegistryEntry: {
         owner: string
         name: string
         url: string (GitHub URL)
         description: string | null
         stars: number
         topics: string[]
         lastAuditDate: string (ISO 8601)
         riskScore: number (0-100)
         findings: AuditFinding[]
         totalFindings: { critical: number, high: number, medium: number, low: number, info: number }
         verifiedSafe: boolean (riskScore === 0)
         scannedFiles: number
         packageName: string | null
       }
       
       Registry: {
         version: string
         lastUpdated: string (ISO 8601)
         totalEntries: number
         entries: RegistryEntry[]
       }

    2. Create lib/constants.ts with SECURITY_RULES array:

       CRITICAL rules (score: 100):
       - CRIT-001: postinstall script containing shell commands (curl|wget|bash|sh|powershell)
         Pattern: /"(?:pre|post)?install"\s*:\s*"[^"]*(?:curl|wget|bash|sh |powershell)[^"]*"/
         fileTargets: ["package.json"]
       - CRIT-002: preinstall script (any content — preinstall is almost never legitimate)
         Pattern: /"preinstall"\s*:\s*"[^"]+"/
         fileTargets: ["package.json"]

       HIGH rules (score: 50):
       - HIGH-001: eval() usage
         Pattern: /\beval\s*\(/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]
       - HIGH-002: child_process require/import
         Pattern: /(?:require|import).*['"]child_process['"]/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]
       - HIGH-003: exec/execSync/spawn usage
         Pattern: /\b(?:exec|execSync|spawn|spawnSync)\s*\(/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]
       - HIGH-004: SSH directory access
         Pattern: /(?:~\/\.ssh|\.ssh\/|id_rsa|id_ed25519|known_hosts)/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]
       - HIGH-005: Environment variable exfiltration (reading sensitive env vars)
         Pattern: /process\.env\.(?:.*TOKEN|.*SECRET|.*KEY|.*PASSWORD|.*CREDENTIAL)/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]

       MEDIUM rules (score: 20):
       - MED-001: Obfuscated Base64 strings (>128 chars)
         Pattern: /['"][A-Za-z0-9+\/=]{128,}['"]/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]
       - MED-002: Unpinned dependencies (using * or latest)
         Pattern: /"\*"|"latest"/
         fileTargets: ["package.json"]
       - MED-003: Network requests to suspicious domains
         Pattern: /(?:raw\.githubusercontent|pastebin\.com|hastebin\.com|transfer\.sh)/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]
       - MED-004: Dynamic require/import (loading code from variables)
         Pattern: /(?:require|import)\s*\(\s*[^'"]/
         fileTargets: ["**/*.ts", "**/*.js", "**/*.mjs"]

       Also export:
       - RISK_THRESHOLDS: { safe: 0, low: 1-19, medium: 20-49, high: 50-79, critical: 80-100 }
       - MAX_RISK_SCORE: 100
       - GITHUB_TOPICS: ["mcp-server", "mcp-tool", "claude-skill", "model-context-protocol"]

    Avoid: Do NOT implement scanning logic — just define the patterns. The auditor (Phase 2) will import these.
  </action>
  <verify>TypeScript compiles without errors: npx tsc --noEmit lib/types.ts lib/constants.ts</verify>
  <done>AC-2 satisfied: Types exported and documented. AC-3 satisfied: All security rules defined with regex patterns.</done>
</task>
