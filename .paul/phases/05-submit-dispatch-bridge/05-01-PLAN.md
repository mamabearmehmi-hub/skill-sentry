<objective>
## Goal
Build the community submission flow: a form where users paste a GitHub URL, an API route that triggers the on-demand-scan GitHub Action via `repository_dispatch`, and integrate it into the dashboard. This is the viral growth engine — when users submit skills, the registry grows automatically.

## Purpose
Without the submit button, Skill Sentry only grows via the daily scraper. The submit button creates a virtuous cycle: users who are scared of a skill come to the site, submit it themselves, get a report, and share the result. Free traffic, free data, community-driven growth.

## Output
- `components/security/SubmitSkillForm.tsx` — URL input form with validation and status feedback
- `app/api/submit/route.ts` — Next.js API route triggering `repository_dispatch` via Octokit
- Updated `app/page.tsx` — submit form embedded in dashboard
- Octokit installed as dependency
</objective>

<acceptance_criteria>

## AC-1: Submit Form Validates GitHub URL
```gherkin
Given the submit form on the dashboard
When a user enters a non-GitHub URL or empty string
Then the form shows a validation error
And does not call the API
When a user enters a valid GitHub URL (https://github.com/owner/repo)
Then the form enables the submit button
```

## AC-2: API Route Triggers repository_dispatch
```gherkin
Given a valid GitHub URL submitted to /api/submit
When the API route processes the request
Then it calls octokit.rest.repos.createDispatchEvent with:
  - owner: the Skill Sentry repo owner (from env)
  - repo: the Skill Sentry repo name (from env)
  - event_type: "scan-repo"
  - client_payload: { repo_url: submitted URL }
And returns 202 Accepted on success
And returns 500 with error message on failure
```

## AC-3: Form Shows Status Feedback
```gherkin
Given a user submits a URL
When the API call is in progress
Then the button shows a loading spinner
When the API returns 202
Then the form shows a success message: "Scan triggered! Results will appear shortly."
When the API returns an error
Then the form shows an error message
```

## AC-4: Form Matches Dashboard Aesthetic
```gherkin
Given the submit form renders on the dashboard
Then it uses the dark industrial theme (dark inputs, teal accents)
And the form is visually prominent (above the table, below stats)
And it matches the JetBrains Mono + Outfit font pairing
```

  <name>Task 1: Install Octokit + create API route</name>
  <files>app/api/submit/route.ts, package.json</files>
  <action>
    1. Install Octokit: `npm install @octokit/rest`

    2. Create app/api/submit/route.ts:
       - Export async POST handler
       - Parse request body: { url: string }
       - Validate URL is a GitHub repo (regex match)
       - Create Octokit instance with process.env.GH_TOKEN
       - Call octokit.rest.repos.createDispatchEvent({
           owner: process.env.GITHUB_OWNER || "mamabearmehmi-hub",
           repo: process.env.GITHUB_REPO || "skill-sentry",
           event_type: "scan-repo",
           client_payload: { repo_url: url }
         })
       - Return NextResponse.json({ status: "triggered" }, { status: 202 })
       - On error: Return NextResponse.json({ error: message }, { status: 500 })

    Environment variables needed (set in Vercel + .env.local):
       - GH_TOKEN: GitHub personal access token with repo + workflow scopes
       - GITHUB_OWNER: repo owner (default: "mamabearmehmi-hub")
       - GITHUB_REPO: repo name (default: "skill-sentry")

    Avoid:
    - Do NOT hardcode the token
    - Do NOT expose the token to the client
    - Do NOT validate the submitted repo exists (let the Action handle that)
  </action>
