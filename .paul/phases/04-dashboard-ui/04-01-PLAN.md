<objective>
## Goal
Build the main dashboard — a searchable, filterable table of all audited MCP skills with a distinctive dark industrial security aesthetic. This is the "Face" of Skill Sentry and the primary marketing surface. It must look like no other AI-built site.

## Purpose
The dashboard is how people discover Skill Sentry. It must communicate trust, authority, and security at first glance. A generic-looking dashboard kills credibility for a security tool. This needs to feel like a command center — dark, precise, slightly ominous.

## Output
- `app/layout.tsx` — dark theme shell with custom fonts and global styling
- `app/page.tsx` — main dashboard page loading registry data
- `app/globals.css` — CSS variables, custom properties, atmospheric effects
- `components/security/` — RiskBadge, ThreatBar, SkillsTable, SearchFilter, StatsHeader
- `lib/registry.ts` — helper to load and sort registry data
</objective>

## AC-1: Dashboard Renders Registry Data
```gherkin
Given registry.json contains 2 entries (safe + malicious)
When I visit the dashboard at localhost:3000
Then I see a table with both entries
And each row shows: repo name, owner, risk score, stars, verified badge, last audit date
```

## AC-2: Risk Score Is Visually Distinct Per Severity
```gherkin
Given entries with different risk scores
When rendered on the dashboard
Then safe (0) shows a calm green/teal badge
And medium (20-49) shows an amber/orange badge
And high (50-79) shows a red badge
And critical (80-100) shows an intense red/pulsing badge
And the visual treatment makes risk level obvious at a glance
```

## AC-3: Search and Filter Works
```gherkin
Given the dashboard table
When I type in the search box
Then the table filters by repo name, owner, or description
And when I toggle "Verified Safe Only"
Then only entries with verifiedSafe=true are shown
```

## AC-4: Dark Industrial Security Aesthetic
```gherkin
Given the dashboard loads
Then the overall theme is dark (near-black background, light text)
And typography uses distinctive fonts (not Inter, Roboto, or system fonts)
And risk indicators use color-coded threat levels
And the layout feels like a security command center, not a generic admin panel
And there is atmospheric depth (gradients, subtle textures, glow effects)
```

## AC-5: Stats Header Shows Aggregate Data
```gherkin
Given the registry has entries
When the dashboard loads
Then a stats header shows: total skills scanned, verified safe count, average risk score
And stats are derived from registry.json at build time
```

    Create components/security/ directory with these components:

    1. RiskBadge.tsx:
       - Props: { score: number; size?: "sm" | "md" | "lg" }
       - Renders the risk score as a styled badge
       - Color maps to RISK_THRESHOLDS: safe=teal, low=green, medium=amber, high=red, critical=pulsing-red
       - Critical scores (80-100) get the pulse-danger animation

    2. ThreatBar.tsx:
       - Props: { findings: { critical: number; high: number; medium: number; low: number; info: number } }
       - Horizontal stacked bar showing finding distribution
       - Each segment colored by severity
       - Zero-findings shows a single teal "CLEAN" bar
       - Compact enough for a table cell

    3. StatsHeader.tsx:
       - Props: { totalScanned: number; verifiedSafe: number; averageRisk: number; criticalCount: number }
       - 4-column stats grid at top of dashboard
       - Each stat: large number + label + subtle icon (Lucide)
       - "Total Scanned" with Shield icon
       - "Verified Safe" with ShieldCheck icon (teal accent)
       - "Avg Risk Score" with AlertTriangle icon (color by level)
       - "Critical Threats" with Skull icon (red accent if > 0)
       - Cards have subtle border glow matching their severity

    4. SearchFilter.tsx:
       - Client component ("use client")
       - Props: { onSearch: (query: string) => void; onFilterSafe: (safe: boolean) => void }
       - Search input with magnifying glass icon (Lucide Search)
       - "Verified Safe Only" toggle button
       - Styled to match dark theme — dark input with subtle border
       - Debounced search (300ms)

    5. SkillsTable.tsx:
       - Client component ("use client")
       - Props: { entries: RegistryEntry[] }
       - Renders the full skills table using Shadcn Table
       - Columns: Repo (owner/name as link), Description, Risk Score (RiskBadge), Findings (ThreatBar), Stars, Last Audit
       - Sortable by risk score (default: highest risk first)
       - Integrates SearchFilter at top
       - Row hover: subtle glow matching risk level
       - Links to /repo/[owner]/[name] (Phase 6 builds the page, link is ready)

    Styling approach:
    - Use Tailwind utility classes + CSS variables from globals.css
    - Use cn() from lib/utils for conditional classes
    - All components respect the dark industrial theme
    - Critical rows should feel slightly menacing
    - Safe rows should feel calm and trustworthy

    Avoid:
    - Do NOT use generic white/gray Shadcn defaults — override everything
    - Do NOT make it look like a typical admin dashboard
    - Keep animations tasteful — one pulse for critical, hover glows, that's it
