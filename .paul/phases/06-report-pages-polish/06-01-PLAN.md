<objective>
## Goal
Build the per-repo detail pages showing the full security audit report with finding cards linked to source code lines, re-enable table row links from the dashboard, and add SEO metadata for sharing. This is the final phase — after this, Skill Sentry is launch-ready.

## Purpose
The detail page is where trust is built or broken. When someone shares "this skill scored 100 on Skill Sentry," the linked report page must show exactly what was found, where, and why it matters. Line-linked source references give the report credibility no other tool offers.

## Output
- `app/repo/[owner]/[name]/page.tsx` — dynamic report page with full audit details
- `components/security/FindingCard.tsx` — individual finding display with severity, file, line link
- `components/security/ReportHeader.tsx` — repo summary header with risk score + metadata
- Updated `components/security/SkillsTable.tsx` — re-enable row links
- Updated `lib/registry.ts` — add `getEntryBySlug` helper
</objective>

```gherkin
Given a registry entry for "shady-dev/mcp-totally-safe"
```

## AC-2: Findings Link to Source Code
```gherkin
Given a finding with file "src/index.ts" and line 3
When displayed on the report page

```

```gherkin
Given a registry entry with riskScore 0 and no findings
When I visit its report page

And no finding cards are rendered
```

```gherkin
Given the dashboard table

And the report page loads with that entry's data
```

## AC-5: SEO Metadata for Sharing
